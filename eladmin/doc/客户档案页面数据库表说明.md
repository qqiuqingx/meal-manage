# 客户档案页面（/customer/profile）数据库表说明

> 本文档梳理 `http://localhost:8013/customer/profile` 页面涉及的所有数据库表及其字段含义。

---

## 涉及的表概览

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| `customer_profile` | 客户档案主档 | 姓名、手机号、孕周、过敏食物、医嘱 |
| `customer_profile_address` | 客户地址 | 默认/工作日/周末三种地址 |
| `customer_profile_package` | 客户签约套餐 | 签约的父子套餐信息 |
| `customer_order` | 客户订单 | 订单金额、餐数、核销进度 |
| `parent_package` | 父套餐字典 | 套餐名称/编码 |
| `sub_package` | 子套餐字典 | 子套餐名称、荤素数量 |
| `parent_package_sub` | 父子套餐关联 | 哪些子套餐属于哪个父套餐（多对多） |

---

## 1. customer_profile — 客户档案主档

主档表，存储客户的核心信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `customer_code` | VARCHAR | 客户编号（如 A001），唯一 |
| `customer_name` | VARCHAR | 客户姓名 |
| `phone` | VARCHAR | 手机号 |
| `gestational_week` | INT | 孕周（正整数，1~50） |
| `allergy_tags` | JSON | 过敏食物标签列表，如 `["虾","花生"]` |
| `medical_requirements` | TEXT | 医嘱要求 |
| `remark` | TEXT | 备注 |
| `create_by` | VARCHAR | 创建人 |
| `update_by` | VARCHAR | 更新人 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

页面展示时还会通过 JOIN 填充以下**非数据库字段**（`@TableField(exist = false)`）：
- `defaultAddress`：默认地址（取 `customer_profile_address` 中 `address_type = 'DEFAULT'` 的第一条）
- `breakfastCount` / `lunchDinnerCount` / `totalCount`：来自最近一条 `customer_order` 的统计
- `parentPackageName` / `childPackageName`：来自最近一条 `customer_order` 的 JOIN

---

## 2. customer_profile_address — 客户地址

一客户多条记录，通过 `address_type` 区分地址类型。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `customer_id` | BIGINT | 关联 `customer_profile.id` |
| `address_type` | VARCHAR | 地址类型：`DEFAULT` / `WORKDAY` / `WEEKEND` |
| `address_detail` | VARCHAR | 详细地址 |
| `contact_name` | VARCHAR | 联系人姓名 |
| `contact_phone` | VARCHAR | 联系人电话 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

页面表单中三条地址对应三个 `address_type`，新建客户时必须包含 DEFAULT 地址（必填），其他两种可选。

---

## 3. customer_profile_package — 客户签约套餐

记录客户的套餐签约信息，每签约一次生成一条记录，`active_flag = 1` 表示当前生效的签约。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `customer_id` | BIGINT | 关联 `customer_profile.id` |
| `parent_package_id` | BIGINT | 关联 `parent_package.id` |
| `child_package_id` | BIGINT | 关联 `sub_package.id` |
| `breakfast_count` | INT | 签约早餐份数 |
| `lunch_dinner_count` | INT | 签约午晚份数 |
| `total_count` | INT | 总份数（后端计算） |
| `start_date` | DATE | 签约开始日期 |
| `end_date` | DATE | 签约结束日期 |
| `active_flag` | TINYINT | 生效标志（1=生效，0=失效） |
| `remark` | TEXT | 备注 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

---

## 4. customer_order — 客户订单

客户每次购买生成的订单记录，贯穿整个送餐周期。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `customer_id` | BIGINT | 关联 `customer_profile.id` |
| `customer_code` | VARCHAR | 冗余存储客户编号 |
| `parent_package_id` | BIGINT | 关联 `parent_package.id` |
| `child_package_id` | BIGINT | 关联 `sub_package.id` |
| `order_code` | VARCHAR | 订单编号，唯一 |
| `deposit_amount` | DECIMAL | 定金金额 |
| `total_amount` | DECIMAL | 标价总金额 |
| `final_amount` | DECIMAL | 实际成交金额 |
| `breakfast_count` | INT | 早餐合计餐数 |
| `lunch_dinner_count` | INT | 午晚合计餐数 |
| `breakfast_price` | DECIMAL | 早餐单价 |
| `lunch_dinner_price` | DECIMAL | 午晚餐单价 |
| `verified_count` | INT | 已核销餐数 |
| `verified_amount` | DECIMAL | 已核销金额 |
| `meal_balance` | DECIMAL | 餐费余额 |
| `remaining_count` | INT | 剩余餐数 |
| `deal_time` | DATETIME | 成交时间 |
| `first_delivery_time` | DATETIME | 第一次送餐时间 |
| `start_date` | DATE | 订单开始日期 |
| `end_date` | DATE | 订单结束日期 |
| `status` | INT | 订单状态：0=已取消，1=进行中，2=已完成 |
| `meal_type` | VARCHAR | 餐次类型：`LUNCH` / `DINNER` / `ALL` |
| `schedule_mode` | VARCHAR | 排餐模式：`SCHEDULE`（指定日期）/ `DAILY`（每天）/ `WEEKEND`（周末）/ `WEEKDAY`（工作日） |
| `delivery_dates` | JSON | 送餐日期列表，如 `["2026-04-01","2026-04-03"]` |
| `customer_source` | VARCHAR | 销售渠道 |
| `remark` | TEXT | 备注 |
| `create_by` | VARCHAR | 创建人 |
| `update_by` | VARCHAR | 更新人 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

详情弹窗中展示的订单列表即来自此表，按 `customer_id` 分页查询。

---

## 5. parent_package — 父套餐字典

套餐大类，如"月子餐"、"孕期餐"等。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `package_code` | VARCHAR | 套餐编码 |
| `prefix` | VARCHAR | 编号前缀（单个大写字母） |
| `package_name` | VARCHAR | 套餐名称 |
| `status` | TINYINT | 状态：1=启用，0=停用 |
| `remark` | TEXT | 备注 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

---

## 6. sub_package — 子套餐字典

套餐小项，包含荤素配比信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `sub_package_code` | VARCHAR | 子套餐编码 |
| `sub_package_name` | VARCHAR | 子套餐名称 |
| `meat_count` | INT | 荤菜数量 |
| `veg_count` | INT | 素菜数量 |
| `include_soup` | TINYINT | 是否含汤：1=是，0=否 |
| `include_rice` | TINYINT | 是否含米饭：1=是，0=否 |
| `status` | TINYINT | 状态：1=启用，0=停用 |
| `remark` | TEXT | 备注 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

---

## 7. parent_package_sub — 父子套餐关联（多对多）

建立父套餐与子套餐的多对多关系。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `parent_package_id` | BIGINT | 关联 `parent_package.id` |
| `sub_package_id` | BIGINT | 关联 `sub_package.id` |
| `status` | INT | 状态 |
| `created_at` | DATETIME | 创建时间 |

页面下拉框中"父套餐 → 子套餐"的树形结构即通过此表 + 两张字典表拼接而来。

---

## 表关系图

```
customer_profile (主档)
├── customer_profile_address (多对一, 通过 customer_id)
├── customer_profile_package (多对一, 通过 customer_id, active_flag=1 取当前签约)
└── customer_order (多对一, 通过 customer_id, 取最新或全部)

parent_package ──(多对多)── sub_package
        ↑                          ↑
        └── parent_package_sub ──┘
```

---

## 新增客户时的数据写入顺序

1. 写入 `customer_profile`，获得 `id`
2. 批量写入 `customer_profile_address`（3条：DEFAULT 必须，WORKDAY/WEEKEND 可选）
3. 写入 `customer_profile_package`（签约记录，`active_flag = 1`）
4. 写入 `customer_order`（首单记录）
