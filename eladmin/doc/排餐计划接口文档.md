# 排餐计划接口文档

## 基础信息

- **基础路径**: `/api/meal-plan`
- **认证方式**: JWT Token (Bearer Token)
- **权限前缀**: `mealPlan:`

## 枚举值说明

### 餐次类型 (meal_type)

| code | desc |
|------|------|
| LUNCH | 午餐 |
| DINNER | 晚餐 |

### 计划状态 (status)

| code | desc |
|------|------|
| GENERATING | 生成中 |
| SUCCESS | 成功 |
| FAILED | 失败（部分成功或全部失败） |

### 客户计划状态 (status)

| code | desc |
|------|------|
| 0 | 失败 |
| 1 | 成功 |

### 菜品类型 (dish_type)

| code | desc |
|------|------|
| MAIN | 主菜 |
| SIDE | 副菜 |
| VEGETABLE | 素菜 |
| SOUP | 汤 |
| RICE | 米饭 |

### 替换原因 (replace_reason)

| code | desc |
|------|------|
| ALLERGY | 过敏回退 |
| NEXT_DAY | 次日回退 |

### 过敏相关字段

| 字段 | 类型 | 说明 |
|------|------|------|
| isAllergyFiltered | Boolean | 是否因过敏被过滤 |
| allergyReasons | String | 触发的过敏标签，多个用逗号分隔（如 "海鲜,豆类"），仅在 isAllergyFiltered=true 时有值 |

## 接口列表

### 1. 生成排餐计划

| 项目 | 说明 |
|------|------|
| **接口地址** | `POST /api/meal-plan/generate` |
| **权限** | `mealPlan:generate` |
| **Content-Type** | `application/json` |

**请求体 (JSON)**:

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| recordDate | String | 是 | 排餐日期，格式 yyyy-MM-dd | 2026-04-02 |
| mealType | String | 是 | 餐次，支持 LUNCH / DINNER | LUNCH |

**请求示例**:
```json
{
  "recordDate": "2026-04-02",
  "mealType": "LUNCH"
}
```

**响应示例**:
```json
{
  "mealPlanId": 1,
  "recordDate": "2026-04-02",
  "mealType": "LUNCH",
  "totalCount": 100,
  "successCount": 98,
  "failCount": 2,
  "failDetails": [
    {
      "failReason": "荤菜不足"
    },
    {
      "failReason": "客户档案不存在"
    }
  ]
}
```

### 5. 删除排餐计划

根据日期和餐次删除排餐计划（级联删除客户和明细）。

| 项目 | 说明 |
|------|------|
| **接口地址** | `DELETE /api/meal-plan` |
| **权限** | `mealPlan:del` |

**请求参数 (Query)**:

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| recordDate | String | 是 | 排餐日期，格式 yyyy-MM-dd | 2026-04-02 |
| mealType | String | 是 | 餐次（LUNCH/DINNER） | LUNCH |

**响应**: `204 No Content`

**示例**:
```
DELETE /api/meal-plan?recordDate=2026-04-02&mealType=LUNCH
```

### 6. 批量删除排餐计划

根据ID列表批量删除排餐计划（级联删除客户和明细）。

| 项目 | 说明 |
|------|------|
| **接口地址** | `DELETE /api/meal-plan/batch` |
| **权限** | `mealPlan:del` |
| **Content-Type** | `application/json` |

**请求体 (JSON)**:

```json
[1, 2, 3]
```

**响应**: `204 No Content`

---

### 7. 批量删除客户排餐计划

根据客户计划ID列表批量删除客户排餐计划（级联删除明细）。

| 项目 | 说明 |
|------|------|
| **接口地址** | `DELETE /api/meal-plan/customers` |
| **权限** | `mealPlan:del` |
| **Content-Type** | `application/json` |

**请求体 (JSON)**:

```json
[1, 2, 3, 4, 5]
```

**响应**: `204 No Content`

### 8. 查询排餐计划客户配送地址

根据排餐计划ID查询其中所有客户的手机号、客户编号和配送地址。

**地址选择规则**（根据排餐计划日期自动选择）：
- 周末（周六/周日）→ `WEEKEND` 地址
- 工作日（周一至周五）→ `WORKDAY` 地址
- 指定类型地址不存在时，fallback 到 `DEFAULT` 地址
- 都没有则返回 null

| 项目 | 说明 |
|------|------|
| **接口地址** | `GET /api/meal-plan/{mealPlanId}/customer-addresses` |
| **权限** | `mealPlan:list` |

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| mealPlanId | Long | 是 | 排餐计划ID |

**响应示例**:
```json
[
  {
    "customerId": 100,
    "customerCode": "A001",
    "phone": "13800138000",
    "addressDetail": "朝阳区XX路XX号",
    "addressType": "WORKDAY"
  },
  {
    "customerId": 101,
    "customerCode": "A002",
    "phone": "13900139000",
    "addressDetail": "海淀区XX大厦XX室",
    "addressType": "DEFAULT"
  }
]
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| customerId | Long | 客户ID |
| customerCode | String | 客户编号 |
| phone | String | 手机号 |
| addressDetail | String | 配送地址（根据日期选定） |
| addressType | String | 实际使用的地址类型：`DEFAULT` / `WORKDAY` / `WEEKEND`，无地址时为 null |

## 权限配置

需要在权限管理中配置以下权限点：

| 权限代码 | 权限名称 | 说明 |
|----------|----------|------|
| `mealPlan:list` | 排餐计划列表 | 查看排餐计划列表和详情 |
| `mealPlan:generate` | 生成排餐计划 | 生成排餐计划 |
| `mealPlan:del` | 删除排餐计划 | 删除排餐计划 |
