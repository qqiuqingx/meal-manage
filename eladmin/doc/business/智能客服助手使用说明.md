# 智能客服助手使用说明

## 运营统计问答

助手支持在当前客服权限范围内查询聚合运营数据，例如“今天待核销客户有多少”“今天已排餐客户有多少”“活跃客户有多少”“未来七天到期订单有多少”。

- “今天还有多少客户”“剩下的客户有多少”“没完成的客户还有几个”等未说明待办口径的问法，都会先询问待排餐、待配送或待核销，避免采用错误口径。
- 支持同日期、同数据源的少量指标组合，例如“今天午餐已排餐和待核销客户分别多少”，也支持“今天待排餐客户按套餐分组”“今天待核销客户按来源分组”。单次最多 3 项、最多两个分组维度，只使用登记指标并复用一次每日工作量聚合；活跃客户、到期订单等不同数据源不会被自动混合成同一报表。
- 统计卡片展示总数、日期、餐次、口径版本、时区和查询时间；不展示客户明细、手机号、地址或金额。
- 日期缺失时，按口径需要补充日期；“今天”“昨天”“本周”等表达按 `Asia/Shanghai` 时区解析。

## 适用对象

仅供已登录且拥有 `agentDiagnosis:list` 及对应业务查询权限的内部客服使用。助手只提供只读查询，不会新增、修改、删除、重排餐、核销或退餐。

## 常用问法

- `B3303 目前什么情况？`
- `B3303 还有多少餐？`
- `B3303 有哪些订单？`
- `订单 O20260711001 什么情况？`

订单卡片会显示关联排餐、核销和退餐记录数量。数量仅用于快速判断是否存在关联业务记录；需要查看具体核销、退餐或排餐内容时，应继续追问对应明细。
- `B3303 今天午餐排了吗？`
- `B3303 今天午餐菜里有什么配料？`
- `B3303 最近核销记录`
- `B3303 最近退过餐吗？`
- `B3303 签了什么套餐？`
- `午餐核销扣哪个池？`
- `B3303 今天午餐已排餐但未核销吗？`
- `今天的菜单是什么？`

## 查询边界

- 支持客户、订单、排餐、核销、退餐、套餐、菜品配料、按客户饮食限制过滤的候选菜预览和已配置业务规则的查询。
- 订单金额、单价、优惠、已收、退款金额和餐费余额均不提供；询问相关内容时会明确拒绝，不会调用金额服务。
- 手机号和配送地址默认脱敏。
- 姓名可能匹配多个客户时，客服需从候选中选择；助手不会自行确定客户。
- 排餐、菜品和历史记录查询需要明确客户、日期或餐次时，助手会追问，避免无界历史查询。
- 查询公共排期菜单时只需日期，不需要客户信息；“今天”“明天”等相对日期会按当前日期解析。

## 权限与异常提示

- 仅有助手入口权限不代表拥有客户、订单、排餐、菜品等数据权限。
- 无权限时，不会以“对象不存在”替代权限错误，也不应尝试通过客户/订单 ID 猜测数据。
- 结果卡片中的“截断”表示安全上限已触发；请缩小日期范围或对象范围后再次查询。
- 查询时间、事实编号和数据来源用于人工复核；关键业务判断以主系统页面和结构化事实为准。

## 排餐诊断

原“排餐未生成诊断”能力仍保留。诊断结论是辅助排查信息，客服应结合证据、客户档案、订单和排餐页面后再作业务处理。
