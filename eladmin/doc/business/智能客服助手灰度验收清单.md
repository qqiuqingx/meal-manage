# 智能客服助手灰度验收清单

## 发布前

- [ ] 主系统和每个 `agent-service` 实例使用相同的 `AGENT_INTERNAL_TOKEN`、`AGENT_ACCESS_CONTEXT_SECRET`，且访问上下文密钥长度不少于 32 字符。
- [ ] Agent 实例可访问主系统内部地址；前端只访问主系统公开接口，不直接访问 Agent 内部接口。
- [ ] 首批仅开放给拥有 `agentDiagnosis:list` 和所需业务只读权限的内部测试客服。
- [ ] 已执行数据库迁移脚本并完成备份验证；金额字段未加入任何 Agent DTO、工具或前端卡片。
- [ ] 已确认 `agent_business_query_audit` 的保留周期和定时清理任务配置。

## 功能抽检

- [ ] 客户概览：`客户编号 + 还有多少餐` 返回脱敏信息、结构化事实和 QueryPlan。
- [ ] 订单、核销、退餐、套餐、排餐、菜单、菜品配料和业务规则问法均只能调用登记工具。
- [ ] 运营统计：待排餐、待核销、已排餐、活跃客户和到期订单均返回口径、时区、查询时间和 facts。
- [ ] 模糊的“今天还有多少客户”必须追问待排餐、待配送或待核销，不能默认执行统计。
- [ ] 金额、写操作、自由 SQL、未登记工具和越权对象请求均被拒绝，且不泄露对象是否存在。

## 多实例与性能

- [ ] 两个 Agent 实例交替处理同一 `sessionId`：实例一识别客户，实例二接收主系统下发的 `contextSlots` 后继续核销或订单查询。
- [ ] 单工具真实请求至少采样 20 次，P95 不超过 3 秒。
- [ ] 多工具组合请求至少采样 20 次，P95 不超过 8 秒。
- [ ] 单工具失败、权限不足和超时均返回受控 `partial=true` 或澄清，不返回异常堆栈、完整地址、手机号或金额。

## 审计与观察

- [ ] 查询审计包含请求、会话、分析来源、指标、维度、工具、耗时、部分失败和回答校验结果。
- [ ] 观察直接回答率、关键澄清率、澄清后成功率、工具失败率、P95、部分回答率、权限拒绝和未支持问题 Top N。
- [ ] 首批客服的异常问法、错误澄清和无用回答均录入反馈或规则缺口处理流程。

## 回滚

- [ ] 停止将流量转发给 Agent 服务即可回退到主系统既有业务页面；Agent 不包含写操作或数据库直连。
- [ ] 若内部查询异常，保留主系统、Agent 与审计日志的 requestId 和 sessionId 用于定位，禁止导出原始访问上下文或敏感工具响应。
- [ ] 灰度结束后，根据审计和反馈决定扩大范围、补充评测案例、调整规则或关闭入口。
