# 排餐生产日期标记实施计划

## 一、需求目标

在排餐计划管理页面 `/customer/scheduleRecord` 的左侧客户编号区域，对临近生产日期的客户进行明显标记。

标记规则：

- 客户生产日期有值。
- 排餐日期大于等于生产日期。
- 排餐日期距离生产日期不超过 3 天。

即以下场景需要标记：

| 排餐日期与生产日期关系 | 是否标记 | 展示说明 |
| --- | --- | --- |
| 排餐日期 = 生产日期 | 是 | 生产当天 |
| 排餐日期 = 生产日期 + 1 天 | 是 | 生产后第 1 天 |
| 排餐日期 = 生产日期 + 2 天 | 是 | 生产后第 2 天 |
| 排餐日期 = 生产日期 + 3 天 | 是 | 生产后第 3 天 |
| 排餐日期早于生产日期 | 否 | 不展示 |
| 排餐日期晚于生产日期 4 天及以上 | 否 | 不展示 |
| 客户生产日期为空 | 否 | 不展示 |

## 二、推荐实现方案

采用后端统一计算标记字段、前端只负责展示的方案。

原因：

- 当前排餐生产单页面依赖 `GET /api/meal-plan/{mealPlanId}/detail` 返回客户列表。
- 该接口当前已返回 `firstMealOfOrder`，前端直接展示“首”标识。
- 页面详情响应中目前没有客户生产日期，若前端计算需要额外按客户查询详情，容易造成接口调用放大。
- 后端在查询排餐计划客户时已经关联 `customer_profile`，可以顺带取 `production_date` 并统一计算。

- 该字段来自 `customer_profile.production_date`，不属于 `meal_plan_customer` 表字段。

### 2. Mapper 查询补充生产日期

文件：

需要调整：

- `BaseResultMap` 增加 `production_date -> productionDate` 映射。
- `selectByMealPlanId` 查询字段补充 `cp.production_date`。
- 如果其他详情接口也复用 `selectByMealPlanIds` 或分页查询展示同类客户信息，可一并补充 `cp.production_date`，保持字段一致。

示例：


```sql

```

### 3. 扩展接口响应 DTO

文件：

- `eladmin/eladmin-system/src/main/java/me/zhengjie/modules/meal/domain/dto/MealPlanDetailVO.java`

在 `CustomerPlanDetail` 中新增字段：

```java
@ApiModelProperty(value = "客户生产日期，格式 yyyy-MM-dd")
private String productionDate;

@ApiModelProperty(value = "是否生产当天至生产后3天内")
private Boolean nearProductionDate;

@ApiModelProperty(value = "排餐日期距离生产日期的天数，生产当天为0")
private Integer productionDateDiffDays;
```

字段说明：

- `productionDate`：用于前端 tooltip 或后续扩展展示。
- `nearProductionDate`：前端是否展示标记的直接判断字段。
- `productionDateDiffDays`：用于展示“生产当天 / 生产后第 N 天”。

### 4. 后端组装逻辑计算标记

文件：

- `eladmin/eladmin-system/src/main/java/me/zhengjie/modules/meal/service/impl/MealPlanServiceImpl.java`

当前 `assembleCustomerDetail(...)` 只接收客户与菜品信息，未直接接收排餐日期。建议调整方法签名，增加 `LocalDate recordDate` 参数。

涉及调用点：

- 组装单个排餐计划详情时。
- 组装排餐计划列表详情时，如复用同一个 `assembleCustomerDetail(...)` 方法。

推荐新增私有方法：

```java
/**
 * 计算排餐日期是否处于客户生产当天至生产后3天内。
 *
 * @param productionDate 客户生产日期
 * @param recordDate 排餐日期
 * @return 生产日期标记信息，生产日期为空或排餐日期为空时返回未命中
 */
private ProductionDateMark buildProductionDateMark(LocalDate productionDate, LocalDate recordDate) {
    if (productionDate == null || recordDate == null) {
        return ProductionDateMark.unmarked();
    }
    long diffDays = ChronoUnit.DAYS.between(productionDate, recordDate);
    boolean marked = diffDays >= 0 && diffDays <= 3;
    return new ProductionDateMark(marked, marked ? (int) diffDays : null);
}
```

也可以不新增内部类，直接在 `assembleCustomerDetail(...)` 中计算：

```java
detail.setProductionDate(customer.getProductionDate() != null ? customer.getProductionDate().toString() : null);
if (customer.getProductionDate() != null && recordDate != null) {
    long diffDays = ChronoUnit.DAYS.between(customer.getProductionDate(), recordDate);
    boolean nearProductionDate = diffDays >= 0 && diffDays <= 3;
    detail.setNearProductionDate(nearProductionDate);
    detail.setProductionDateDiffDays(nearProductionDate ? (int) diffDays : null);
} else {
    detail.setNearProductionDate(false);
    detail.setProductionDateDiffDays(null);
}
```

注意：

- 需要引入 `java.time.temporal.ChronoUnit`。
- 生产日期标记只对展示生效，不参与排餐资格、菜品生成、核销、统计计算。
- 不建议因为生产日期标记调整客户排序，避免改变现有生产单顺序。

## 四、前端改造计划

### 1. 左侧客户编号区域增加标记

文件：

- `eladmin-web/src/views/meal/scheduleRecord/index.vue`

当前客户编号区域已有首单标记：

```vue
<span v-if="customer.firstMealOfOrder" class="code-first-badge">首</span>
```

建议新增生产日期标记：

```vue
<el-tooltip
  v-if="customer.nearProductionDate"
  effect="dark"
  placement="top"
  :content="getProductionDateBadgeTip(customer)"
>
  <span class="code-production-badge">产</span>
</el-tooltip>
```

如果客户同时是首单、又命中生产日期标记，需要两个标记同时展示。

### 2. 增加 tooltip 文案方法

文件：

- `eladmin-web/src/views/meal/scheduleRecord/index.vue`

新增方法：

```js
getProductionDateBadgeTip(customer) {
  const days = Number(customer.productionDateDiffDays)
  if (days === 0) {
    return '生产当天'
  }
  if (days > 0) {
    return `生产后第${days}天`
  }
  return '临近生产日期'
}
```

如需展示具体日期，可扩展为：

```js
return `${text}，生产日期：${customer.productionDate}`
```

### 3. 调整徽标布局

当前 `.code-first-badge` 使用绝对定位：

```css
.code-first-badge {
  position: absolute;
  top: 10px;
  right: 10px;
}
```

如果新增第二个徽标，建议引入统一容器，避免两个徽标重叠：

```vue
<div v-if="customer.firstMealOfOrder || customer.nearProductionDate" class="code-badges">
  <span v-if="customer.firstMealOfOrder" class="code-badge code-badge--first">首</span>
  <el-tooltip v-if="customer.nearProductionDate" effect="dark" placement="top" :content="getProductionDateBadgeTip(customer)">
    <span class="code-badge code-badge--production">产</span>
  </el-tooltip>
</div>
```

- 移动端打印样式中也要检查徽标大小，避免遮挡客户编号。

### 4. 排序规则保持不变

当前 `allCustomers` 计算属性按 `firstMealOfOrder` 将首餐客户前置：

```js
const firstCustomers = decorated.filter(item => item.firstMealOfOrder)
const normalCustomers = decorated.filter(item => !item.firstMealOfOrder)
return [...firstCustomers, ...normalCustomers]
```

生产日期标记不参与排序。

如果后续业务要求生产日期客户也前置，需要单独确认优先级，例如：

1. 首餐 + 生产日期
2. 首餐
3. 生产日期
4. 普通客户

本次不建议调整排序。

## 八、联调验收清单

1. 选择一个客户，设置生产日期为排餐日期当天，进入 `/customer/scheduleRecord`，客户编号右上角展示“产”。
2. 设置生产日期为排餐日期前 1、2、3 天，均展示“产”。
3. 设置生产日期为排餐日期前 4 天，不展示“产”。
4. 设置生产日期晚于排餐日期，不展示“产”。
5. 客户同时为首餐客户时，编号区域同时展示“首”和“产”，且不重叠。
6. 鼠标悬停“产”标记，tooltip 展示正确天数。
7. 客户排序仍保持现有首餐优先逻辑，生产日期标记不改变排序。
8. 打印页面检查徽标不遮挡客户编号和特殊要求标签。
9. 接口响应中 `customers` 数组包含新增字段，旧字段兼容不变。
10. 业务文档和接口文档已同步更新。

## 九、风险点与注意事项

- `production_date` 是客户档案字段，不应落入 `meal_plan_customer` 表，避免历史排餐数据和客户档案产生双写不一致。
- 标记应按当前客户档案生产日期实时计算；如果业务要求历史生产单保持生成当时的生产日期，需要另行设计快照字段。
- 该标识只做视觉提醒，不应影响排餐、换菜、核销、统计等业务逻辑。
- 新增两个徽标后，要重点检查左侧编号区域宽度较窄时的换行和遮挡问题。
- 如果生产日期标记未来也要出现在 `/meal/productionSheet` 页面，应复用同一接口字段和前端样式。
