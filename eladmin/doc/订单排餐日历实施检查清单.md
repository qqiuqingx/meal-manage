## 使用步骤

### 启动/重启开发服务器

```bash
cd eladmin-web
# 停止当前运行的 dev 服务器（如果正在运行）
# Ctrl+C

# 启动开发服务器
npm run dev
```

### 访问订单管理页面

1. 登录系统后，进入"订单管理"
2. 点击"新增订单"或"编辑"按钮
3. 在"排餐模式"下拉框中选择"指定日期送"
4. 日历组件应该会显示在"送餐日期"区域

## 常见问题排查

### 问题1: 选择"指定日期送"后没有显示日历

**可能原因**:
1. 开发服务器未重启
2. 浏览器缓存未刷新
3. scheduleMode 值不匹配

**解决方法**:
```bash
# 1. 停止并重启开发服务器
cd eladmin-web
npm run dev

# 2. 清除浏览器缓存
# Chrome: Ctrl+Shift+R (强制刷新)
# Mac: Cmd+Shift+R

# 3. 检查浏览器控制台是否有报错
# F12 打开开发者工具，查看 Console 标签
```

### 问题2: Vue 警告提示函数未定义

**已修复**: 已将 `isDateInRange` 重命名为 `checkDateInRange` 并添加 `isDateValid` 方法

### 问题3: 点击日期没有反应

**检查步骤**:
1. 检查 `startDate` 和 `endDate` 是否已设置
2. 检查点击的日期是否在订单日期范围内
3. 检查浏览器控制台是否有错误

### 问题4: 餐数没有同步到订单表单

**检查**:
- OrderForm.vue 中的 `onCalendarSelectionChange` 方法是否被调用
- 查看 computed 属性 `deliveryDatesWithMealTypes` 的 getter/setter 是否正常
