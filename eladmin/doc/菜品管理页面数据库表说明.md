# 菜品管理页面（/meal/dish）数据库表说明

> 本文档梳理 `http://localhost:8013/meal/dish` 页面涉及的所有数据库表及其字段含义。

---

## 涉及的表概览

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| `dish` | 菜品主档 | 名称、类型、做法、图片、餐次、套餐、排期 |
| `dish_ingredient` | 配料字典 | 配料名称、分类、单位、热量 |
| `dish_ingredient_relation` | 菜品-配料关联 | 菜品ID、配料ID、用量、备注 |

---

## 1. dish — 菜品主档

核心表，存储所有菜品信息，多个 JSON 字段用于存储列表/枚举值。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键，自增 |
| `name` | VARCHAR | 菜品名称，必填 |
| `cooking_method` | TEXT | 做法/流程 |
| `ingredients` | TEXT | 配料（冗余文本字段，备用） |
| `image_url` | VARCHAR | 图片路径 |
| `dish_type` | VARCHAR | 菜品类型：`MAIN` 主菜 / `SIDE` 副菜 / `SOUP` 汤 / `VEGETABLE` 素菜 / `RICE` 米饭 |
| `meal_types` | JSON Array | 适用餐次：`["LUNCH"]` / `["DINNER"]` / `["LUNCH","DINNER"]` |
| `meal_packages` | JSON Array | 所属套餐代码：`["yuezi"]` / `["yunqi","yingyang"]` 等，存枚举 code |
| `schedule` | JSON Array | 排期，格式 `["1-1","2-3"]` 表示"第1周周一"、"第2周周三" |
| `sort` | INT | 排序值，越小越靠前 |
| `enabled` | TINYINT | 是否启用：true=启用，false=禁用 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

### JSON 字段详解

**`schedule` 格式**

每项为 `"W-D"` 格式，W=周序号（1-4），D=星期几（1=周一，7=周日）。

```
["1-1", "2-1", "3-1", "4-1"]  →  每周一出现
["3-5"]                       →  第3周周五出现
```

**`meal_packages` 值域**

来自 `MealPackageEnum` 枚举：

| code | 含义 |
|------|------|
| `yuezi` | 月子餐 |
| `yunqi` | 孕期餐 |
| `xiaoyuezi` | 小月子 |
| `yingyang` | 营养餐 |
| `fenmian` | 分娩餐 |

页面表单中"套餐"下拉框展示的是 `parent_package` 表数据（父子套餐结构），与 `meal_packages` 的 JSON code 是**两套独立的体系**：前者用于客户订单模块，后者用于菜品所属的餐品线分类。

**非数据库字段（`@TableField(exist = false)`）**

- `ingredientList`：菜品关联的配料详情列表（来自 `dish_ingredient_relation` JOIN 查询）
- `mealPackageDetails`：套餐详情（来自 `parent_package` 表 JOIN，用于表格展示）

---

## 2. dish_ingredient — 配料字典

菜品的配料主档，类似"食材字典"，支持按名称搜索、分类筛选。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键，自增 |
| `name` | VARCHAR | 配料名称，必填，唯一 |
| `category` | VARCHAR | 配料分类：`MEAT` 肉类 / `VEGETABLE` 蔬菜 / `SEAFOOD` 海鲜 / `TOFU` 豆制品 / `SPICE` 调料 / `OTHER` 其他 |
| `unit` | VARCHAR | 单位：`g` 克 / `ml` 毫升 / `个` |
| `calories` | INT | 每单位热量（卡路里） |
| `remark` | TEXT | 备注 |
| `enabled` | TINYINT | 是否启用（配料搜索默认只返回 enabled=true 的记录） |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

页面表单中，配料通过远程搜索框（remote method）查询此表，选择后添加到菜品的配料列表中。

---

## 3. dish_ingredient_relation — 菜品-配料关联

菜品与配料的多对多关系表，记录每个菜品用了哪些配料、各自用量多少。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT | 主键，自增 |
| `dish_id` | INT | 关联 `dish.id` |
| `ingredient_id` | INT | 关联 `dish_ingredient.id` |
| `ingredient_name` | VARCHAR | 冗余存储配料名称（避免查配料表） |
| `unit` | VARCHAR | 单位（冗余存储） |
| `quantity` | DOUBLE | 用量（与 `unit` 配合，如 100g） |
| `remark` | TEXT | 备注（如"少盐"、"切丁"） |
| `create_time` | DATETIME | 创建时间 |

页面表单中点击"添加配料"后，`ingredientList`（前端内存数组）保存的每条记录最终写入此表；编辑时从此表读取并回填。

---

## 表关系图

```
dish                              ← 菜品主档
 ├── meal_packages (JSON)          ← 直接存套餐代码（yuezi/yunqi...）
 └── dish_ingredient_relation (1对多) ← 关联 dish_ingredient

dish_ingredient                    ← 配料字典（可复用）
```

---

## 页面表单与表的对应关系

| 表单字段 | 对应表/字段 | 存储方式 |
|----------|------------|----------|
| 菜品名称 | `dish.name` | 直接写入 |
| 菜品类型 | `dish.dish_type` | 直接写入 |
| 做法/流程 | `dish.cooking_method` | 直接写入 |
| 图片路径 | `dish.image_url` | 直接写入 |
| 餐次 | `dish.meal_types` | JSON 数组写入 |
| 套餐 | `dish.meal_packages` | JSON 数组写入（套餐代码） |
| 排期 | `dish.schedule` | JSON 数组写入 |
| 排序 | `dish.sort` | 直接写入 |
| 是否启用 | `dish.enabled` | 直接写入 |
| 配料列表 | `dish_ingredient_relation` | 每条配料 INSERT 一行 |
| 配料搜索 | `dish_ingredient` | 读，remote search |
