# 父套餐餐数统计接口

## 接口信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 查询指定日期的各个父套餐的餐数 |
| 请求方式 | GET |
| 接口路径 | `/api/meal-plan/statistics-by-date` |
| 权限 | `mealPlan:list` |

## 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| date | String | 是 | 排餐日期，格式 yyyy-MM-dd |

## 请求示例

```http
GET /api/meal-plan/statistics-by-date?date=2026-04-05
```

## 返回结果

```json
[
  {
    "packageCode": "YZ",
    "packageName": "月子餐",
    "mealCount": 10
  },
  {
    "packageCode": "YYC001",
    "packageName": "营养餐",
    "mealCount": 5
  },
  {
    "packageCode": "JKC001",
    "packageName": "健康餐 2",
    "mealCount": 8
  }
]
```

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| packageCode | String | 套餐代码 |
| packageName | String | 套餐名称 |
| mealCount | Integer | 餐数 |

## 数据来源

- `meal_plan` 表（排餐计划主表）
- `meal_plan_customer` 表（排餐计划客户表）
- `parent_package` 表（父套餐信息表）

## 查询逻辑

按日期分组统计各父套餐的客户数量（即餐数），按餐数降序排列。

```sql
SELECT
    pp.package_code,
    pp.package_name,
    COUNT(mpc.id) AS meal_count
FROM meal_plan mp
JOIN meal_plan_customer mpc ON mp.id = mpc.meal_plan_id
JOIN parent_package pp ON mpc.parent_package_id = pp.id
WHERE mp.record_date = #{date}
  AND mp.deleted = 0
  AND mpc.deleted = 0
GROUP BY pp.id, pp.package_code, pp.package_name
ORDER BY meal_count DESC
```
