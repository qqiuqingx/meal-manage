# 核销管理接口文档

## 基础信息

- **基础路径**: `/api/meal-verification`
- **认证方式**: JWT Token (Bearer Token)
- **权限前缀**: `mealPlan:` / `verificationLog:`

---

## 接口列表

### 1. 执行核销

| 项目 | 说明 |
|------|------|
| **接口地址** | `POST /api/meal-verification/verify` |
| **权限** | `mealPlan:list` |
| **Content-Type** | `application/json` |

**请求体 (JSON)**:

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| customerPlanIds | Array[Long] | 是 | 客户排餐ID列表 | [1, 2, 3] |
| remark | String | 否 | 备注 | 批量核销 |

**请求示例**:
```json
{
  "customerPlanIds": [1, 2, 3],
  "remark": "批量核销"
}
```

**响应示例**:
```json
{
  "successCount": 2,
  "failCount": 1,
  "failReasons": [
    "客户ID 3: 订单剩余餐数不足，当前剩余: 0"
  ]
}
```

## 业务说明

### 核销流程

1. **校验客户排餐记录是否存在**
2. **检查是否已核销**（已核销则不允许重复核销）
3. **检查订单剩余餐数是否足够**（剩余餐数 > 0）
4. **计算核销单价**：
   - 午餐/晚餐：使用订单的 `lunch_dinner_price`
   - 早餐：使用订单的 `breakfast_price`
5. **原子性更新订单**：
   - `verified_count` + 1（已核销餐数）
   - `verified_amount` + 单价（核销金额）
   - `meal_balance` - 单价（餐费余额）
   - `remaining_count` - 1（剩余餐数）
6. **自动完成订单**：
   - 如果订单剩余餐数（`remaining_count`）减至 0，系统自动将订单状态更新为"已完成"（status = 2）
   - 此更新在核销事务中原子性执行，保证数据一致性
   - 只有状态为"进行中"（status = 1）的订单才会被自动完成
7. **更新客户排餐核销状态**：
   - `is_verified` = 1
   - `verification_time` = 当前时间
   - `verification_operator` = 当前操作人
8. **记录核销日志**到 `meal_verification_log` 表

#### 自动完成订单
- 当订单剩余餐数（remaining_count）减至 0 时，系统自动将订单状态更新为"已完成"（status = 2）
- 此更新在核销事务中原子性执行，保证数据一致性
- 只有状态为"进行中"（status = 1）的订单才会被自动完成
- 已取消（status = 0）或已完成（status = 2）的订单不受影响

### 核销失败原因

| 错误信息 | 说明 |
|----------|------|
| 客户排餐记录不存在 | 客户排餐ID无效 |
| 该客户已完成核销，请勿重复操作 | 该客户已被核销过 |
| 关联订单不存在 | 订单数据异常 |
| 订单剩余餐数不足 | 订单没有剩余餐数可核销 |
| 订单单价配置异常，无法核销 | 订单的 lunchDinnerPrice 或 breakfastPrice 为空或≤0 |
| 订单更新失败 | 并发冲突或其他数据库错误 |

### 数据库字段说明

#### meal_verification_log（核销日志表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| meal_plan_customer_id | BIGINT | 客户排餐ID |
| meal_plan_id | BIGINT | 排餐计划ID |
| customer_id | BIGINT | 客户ID |
| order_id | BIGINT | 关联订单ID |
| record_date | DATE | 排餐日期 |
| meal_type | VARCHAR(20) | 餐次（LUNCH/DINNER） |
| verification_count | INT | 核销餐数（默认1） |
| remaining_before | INT | 核销前剩余餐数 |
| remaining_after | INT | 核销后剩余餐数 |
| verified_total_before | INT | 核销前已核销总数 |
| verified_total_after | INT | 核销后已核销总数 |
| operator | VARCHAR(50) | 操作人 |
| operate_time | DATETIME | 操作时间 |
| remark | VARCHAR(255) | 备注 |

#### meal_plan_customer（排餐客户表）新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| is_verified | TINYINT(1) | 是否已核销（0未核销，1已核销） |
| verification_time | DATETIME | 核销时间 |
| verification_operator | VARCHAR(50) | 核销操作人 |
