# 客户订单管理 API 文档

> 文档版本: 1.0
> 创建日期: 2026-03-26

## 1. 概述

客户订单管理模块提供订单的创建、编辑、删除、查询等能力，订单关联客户档案。

### 1.1 模块结构

- **后端路径**: `/api/customer/order`
- **前端路径**: `customer/order/index`

### 1.2 权限定义

| 权限标识 | 说明 |
|---------|------|
| `customerOrder:list` | 订单列表查询 |
| `customerOrder:add` | 新增订单 |
| `customerOrder:edit` | 编辑订单 |
| `customerOrder:del` | 删除订单 |

### 1.3 订单状态

| 状态值 | 说明 |
|-------|------|
| 0 | 已取消 |
| 1 | 进行中 |
| 2 | 已完成 |

### 2.3 新增订单

**请求**

```
POST /api/customer/order
Content-Type: application/json
```

**请求体**

```json
{
  "customerId": 1,
  "parentPackageId": 1,
  "childPackageId": 2,
  "depositAmount": 1000.00,
  "totalAmount": 5000.00,
  "finalAmount": 4800.00,
  "breakfastCount": 30,
  "lunchDinnerCount": 60,
  "breakfastPrice": 25.00,
  "lunchDinnerPrice": 45.00,
  "verifiedCount": 0,
  "verifiedAmount": 0.00,
  "mainDishCount": 2,
  "sideDishCount": 1,
  "vegCount": 3,
  "riceCount": 1,
  "riceType": "白米饭",
  "soupCount": 1,
  "dealTime": "2026-03-26T10:00:00",
  "firstDeliveryTime": "2026-04-01T07:30:00",
  "startDate": "2026-04-01",
  "endDate": "2026-06-30",
  "status": 1,
  "remark": ""
}
```

**字段说明**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerId | Long | 是 | 客户ID |
| parentPackageId | Long | 否 | 父套餐ID |
| childPackageId | Long | 否 | 子套餐ID |
| totalAmount | BigDecimal | 是 | 总金额 |
| finalAmount | BigDecimal | 是 | 成交金额 |
| depositAmount | BigDecimal | 否 | 定金金额，默认 0 |
| breakfastCount | Integer | 否 | 早餐合计餐数，默认 0 |
| lunchDinnerCount | Integer | 否 | 午餐+晚餐合计餐数，默认 0 |
| breakfastPrice | BigDecimal | 否 | 早餐单价，默认 0 |
| lunchDinnerPrice | BigDecimal | 否 | 午餐晚餐单价，默认 0 |
| verifiedCount | Integer | 否 | 核销餐数，默认 0 |
| verifiedAmount | BigDecimal | 否 | 核销金额，默认 0 |
| mainDishCount | Integer | 是 | 每餐主菜数量，>= 0 |
| sideDishCount | Integer | 是 | 每餐副菜数量，>= 0 |
| vegCount | Integer | 是 | 每餐素菜数量，>= 0 |
| riceCount | Integer | 否 | 每餐米饭数量，默认 1 |
| riceType | String | 否 | 米饭类型，默认 `白米饭`；建议从 `GET /api/dishes?dishType=RICE_TYPE&enabled=true` 动态查询菜品名称后回填 |
| soupCount | Integer | 是 | 每餐汤数量，>= 0 |
| dealTime | LocalDateTime | 否 | 成交时间 |
| firstDeliveryTime | LocalDateTime | 否 | 第一次送餐时间 |
| startDate | LocalDate | 否 | 订单开始日期 |
| endDate | LocalDate | 否 | 订单结束日期 |
| status | Integer | 否 | 订单状态，默认 1（进行中） |
| remark | String | 否 | 备注 |

**业务规则**

- `finalAmount` 不能超过 `totalAmount`
- `verifiedCount` 不能超过合计餐数
- `verifiedAmount` 不能超过 `finalAmount`
- `endDate` 不能早于 `startDate`
- `餐费余额` = `成交金额` - `核销金额`（后端自动计算）
- `剩余餐数` = 合计餐数 - 核销餐数（后端自动计算）
- `订单编号` 自动生成，格式：`ORDyyyyMMddXXX`（XXX为当日序号）
- `riceCount` 未传时默认 `1`，`riceType` 未传时默认 `白米饭`
- `riceType` 建议提交 `dish_type = RICE_TYPE` 的菜品名称，订单页可通过菜品查询接口动态加载

**响应**

```
201 Created
```

### 2.4 编辑订单

**请求**

```
PUT /api/customer/order
Content-Type: application/json
```

**请求体**

同 2.3 新增订单，需额外传入 `id` 字段。

**业务规则**

- 只有在编辑时修改了 `parentPackageId`，系统才会按新父套餐编号池刷新**当前订单**的 `customerCode`
- 上述刷新会同步回写 `customer_profile.customer_code`，但不会改动该客户的其他订单

---

### 2.5 删除订单

**请求**

```
DELETE /api/customer/order
Content-Type: application/json
```

**请求体**

```json
[1, 2, 3]
```

**响应**

```
200 OK
```

## 3. 数据库表结构

```sql
CREATE TABLE customer_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL,
    customer_code VARCHAR(32),
    parent_package_id BIGINT,
    child_package_id BIGINT,
    order_code VARCHAR(32) NOT NULL,
    deposit_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    total_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    final_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    breakfast_count INT NOT NULL DEFAULT 0,
    lunch_dinner_count INT NOT NULL DEFAULT 0,
    breakfast_price DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    lunch_dinner_price DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    verified_count INT NOT NULL DEFAULT 0,
    verified_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    meal_balance DECIMAL(10,2) NOT NULL DEFAULT 0.00,
    remaining_count INT NOT NULL DEFAULT 0,
    main_dish_count INT NOT NULL DEFAULT 0,
    side_dish_count INT NOT NULL DEFAULT 0,
    veg_count INT NOT NULL DEFAULT 0,
    rice_count INT NOT NULL DEFAULT 1,
    rice_type VARCHAR(20) DEFAULT '白米饭',
    soup_count INT NOT NULL DEFAULT 0,
    deal_time DATETIME,
    first_delivery_time DATETIME,
    start_date DATE,
    end_date DATE,
    status TINYINT NOT NULL DEFAULT 1,
    meal_type VARCHAR(32),
    schedule_mode VARCHAR(32),
    delivery_dates VARCHAR(500),
    customer_source VARCHAR(64),
    remark VARCHAR(255),
    create_by VARCHAR(100),
    update_by VARCHAR(100),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```
