# 退餐管理接口

## 接口概述

退餐管理用于处理客户取消订单的场景，退餐 = 取消订单 + 取消排餐。

## 业务规则

- 只有 `status=1`（进行中）的订单可以退餐
- 退款金额 = 剩余餐数 × 单价（只退未核销部分）
- 退餐后订单状态变为 `status=3`（已退餐）
- 核销日志保留，标记 `is_refunded=1`

## API 接口

### 1. 执行退餐

**请求信息**

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/meal-refund` |
| 认证方式 | JWT Token |
| 权限验证 | `mealPlan:list` |

**请求参数**

```json
{
  "orderId": 26,
  "refundReason": "客户要求取消订单"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| orderId | Long | 是 | 订单ID |
| refundReason | String | 是 | 退餐原因 |

**响应示例**

```json
{
  "id": 1,
  "orderId": 26,
  "customerId": 5,
  "refundAmount": 2700.00,
  "refundBreakfastCount": 0,
  "refundLunchDinnerCount": 30,
  "verifiedBreakfastCount": 0,
  "verifiedLunchDinnerCount": 0,
  "refundReason": "客户要求取消订单",
  "operator": "admin",
  "operateTime": "2026-04-19T15:30:00",
  "createTime": "2026-04-19T15:30:00"
}
```

**响应字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 退餐日志ID |
| orderId | Long | 订单ID |
| customerId | Long | 客户ID |
| refundAmount | BigDecimal | 退款金额 |
| refundBreakfastCount | Integer | 退早餐数 |
| refundLunchDinnerCount | Integer | 退午晚餐数 |
| verifiedBreakfastCount | Integer | 已核销早餐数（不退） |
| verifiedLunchDinnerCount | Integer | 已核销午晚餐数（不退） |
| refundReason | String | 退餐原因 |
| operator | String | 操作人 |
| operateTime | Date | 操作时间 |
| createTime | Date | 创建时间 |

**错误码**

| HTTP状态码 | 错误信息 | 说明 |
|------------|----------|------|
| 400 | 订单不存在 | 传入的订单ID不存在 |
| 400 | 只有进行中的订单可以退餐 | 订单状态不是进行中 |

### 2. 分页查询退餐日志

**请求信息**

| 项目 | 内容 |
|------|------|
| 接口路径 | `GET /api/meal-refund/logs` |
| 认证方式 | JWT Token |
| 权限验证 | `mealPlan:list` |

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerName | String | 否 | 客户姓名（模糊查询） |
| orderId | Long | 否 | 订单ID |
| operateTimeStart | String | 否 | 操作时间开始，格式：yyyy-MM-dd |
| operateTimeEnd | String | 否 | 操作时间结束，格式：yyyy-MM-dd |
| operator | String | 否 | 操作人 |
| page | Integer | 否 | 页码，默认 0 |
| size | Integer | 否 | 每页数量，默认 10 |

**响应示例**

```json
{
  "content": [
    {
      "id": 1,
      "orderId": 26,
      "customerId": 5,
      "refundAmount": 2700.00,
      "refundBreakfastCount": 0,
      "refundLunchDinnerCount": 30,
      "verifiedBreakfastCount": 0,
      "verifiedLunchDinnerCount": 0,
      "refundReason": "客户要求取消订单",
      "operator": "admin",
      "operateTime": "2026-04-19T15:30:00",
      "createTime": "2026-04-19T15:30:00",
      "customerName": "张三",
      "customerCode": "A1001",
      "orderCode": "ORD20260419001"
    }
  ],
  "totalElements": 1,
  "totalPages": 1
}
```

**响应字段说明**

继承退餐日志所有字段，新增：

| 字段 | 类型 | 说明 |
|------|------|------|
| customerName | String | 客户姓名 |
| customerCode | String | 客户编号 |
| orderCode | String | 订单编号 |

| customer_id | BIGINT | 客户ID |
| refund_amount | DECIMAL | 退款金额 |
| refund_breakfast_count | INT | 退早餐数 |
| refund_lunch_dinner_count | INT | 退午晚餐数 |
| verified_breakfast_count | INT | 已核销早餐数（不退） |
| verified_lunch_dinner_count | INT | 已核销午晚餐数（不退） |
| refund_reason | VARCHAR | 退餐原因 |
| operator | VARCHAR | 操作人 |
| operate_time | DATETIME | 操作时间 |
| create_time | DATETIME | 创建时间 |
