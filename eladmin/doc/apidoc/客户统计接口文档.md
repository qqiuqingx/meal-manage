# 客户统计接口

## customer-stats

获取当天客户总数，按套餐和餐次分组统计。

### 请求

```
GET /api/dishes/schedule/customer-stats?date=2026-04-05
```

### 响应

```json
{
  "date": "2026-04-05",
  "totalCustomerCount": 10,
  "groups": [
    { "mealType": "LUNCH", "mealPackage": "yuezi", "mealPackageDesc": "月子餐", "customerCount": 5 },
    { "mealType": "LUNCH", "mealPackage": "yunqi", "mealPackageDesc": "孕期餐", "customerCount": 3 },
    { "mealType": "DINNER", "mealPackage": "yuezi", "mealPackageDesc": "月子餐", "customerCount": 2 }
  ],
  "sourceGroups": [
    { "source": "1380013", "sourceDesc": "1380013", "customerCount": 3 },
    { "source": "1390009", "sourceDesc": "1390009", "customerCount": 7 }
  ]
}
```

### 数据来源

| 字段 | 来源表 |
|------|-------|
| 排餐计划 | meal_plan |
| 客户排餐 | meal_plan_customer |
| 套餐 | customer_order + parent_package |
| 来源标识 | customer_profile.phone 前7位 |

## customer-meal-stats

分页查询客户用餐统计，并返回查询月份内该客户需要排餐的具体日期。

### 请求

```
GET /api/customerProfile/mealStats?statsMonth=2026-04&page=1&size=20
```

### 查询参数

| 参数 | 必填 | 说明 |
|------|------|------|
| customerCode | 否 | 客户编号 |
| customerName | 否 | 客户姓名 |
| phone | 否 | 手机号 |
| statsMonth | 否 | 统计月份，格式 `yyyy-MM`；为空时排餐日历按当前月份计算 |
| page | 否 | 页码，默认 1 |
| size | 否 | 每页条数，默认 10 |

### 响应字段

```json
{
  "content": [
    {
      "rowKey": "1-LUNCH_DINNER",
      "customerId": 1,
      "orderId": 1001,
      "customerCode": "A001",
      "phone": "13800138000",
      "deliveryInfo": "工作日/午餐/晚餐",
      "mealCount": 20,
      "remainingMealCount": 18,
      "mealBucket": "LUNCH_DINNER",
      "scheduleDays": [
        { "date": "2026-04-01", "mealTypes": ["LUNCH", "DINNER"] },
        { "date": "2026-04-02", "mealTypes": ["LUNCH"] }
      ],
      "baseScheduleDays": [
        { "date": "2026-04-02", "mealTypes": ["LUNCH", "DINNER"] }
      ],
      "customerScheduleDays": [
        {
          "date": "2026-04-01",
          "mealTypes": ["BREAKFAST", "LUNCH", "DINNER"],
          "baseMealTypes": ["BREAKFAST", "LUNCH", "DINNER"],
          "excludedMealTypes": [],
          "addedMealTypes": ["DINNER"],
          "scheduledMealTypes": ["LUNCH"]
        },
        {
          "date": "2026-04-02",
          "mealTypes": ["BREAKFAST", "LUNCH"],
          "baseMealTypes": ["BREAKFAST", "LUNCH", "DINNER"],
          "excludedMealTypes": ["DINNER"],
          "addedMealTypes": [],
          "scheduledMealTypes": []
        }
      ]
    }
  ],
  "totalElements": 1
}
```

### 排餐日历字段

- `scheduleDays`：当前统计行对应餐数桶的排餐日期，`BREAKFAST` 行只含早餐，`LUNCH_DINNER` 行只含午餐/晚餐。
- `baseScheduleDays`：不应用 `excluded_dates` 的基础应排餐日期，用于前端展示可恢复餐次。
- `customerScheduleDays`：同一个客户的早餐和午晚餐排餐日期合并结果，用于页面每个客户一个“排餐日历”按钮。
- `baseMealTypes`：该日期来自订单基础排餐规则的餐次。
- `excludedMealTypes`：该日期被 `customer_profile.exclude_dates` 取消的餐次。
- `addedMealTypes`：该日期来自 `customer_meal_schedule_addition` 的人工新增餐次。
- `scheduledMealTypes`：该日期已生成排餐记录的餐次，前端在对应餐次标签上显示勾选标记。

### 计算规则

- 按 `statsMonth` 展开当月日期，并限制在订单 `start_date` 到 `end_date` 范围内。
- 若日期等于 `start_date`，按 `start_meal_type` 过滤未开送餐次。
- 按订单 `meal_type` 过滤餐次。
- 按订单 `schedule_mode` 过滤日期：`DAILY` 每日，`WEEKDAY` 工作日，`WEEKEND` 周末，`SCHEDULE` 按 `delivery_dates` 中的日期和餐次。
- 按客户档案 `excluded_dates` 排除指定日期和餐次。

## customer-meal-schedule-adjustments

保存客户排餐日历调整。该接口只保存日历状态，不立即生成新的排餐计划。

### 请求

```
PUT /api/customerProfile/mealStats/scheduleAdjustments
```

### 请求体

```json
{
  "customerId": 1,
  "excludedDates": [
    { "date": "2026-04-02", "mealTypes": ["DINNER"] }
  ],
  "additions": [
    { "orderId": 1001, "date": "2026-04-05", "mealType": "LUNCH", "remark": "临时加餐" }
  ]
}
```

### 响应

```json
{
  "customerId": 1,
  "excludedMealCount": 1,
  "additionMealCount": 1,
  "deletedUnverifiedPlanCount": 1
}
```

### 规则

- `excludedDates` 和 `additions` 都按页面保存后的完整列表提交。
- 新增排除餐次时，如果已有未核销排餐记录，会同步软删除该客户排餐记录和明细。
- 新增排除餐次时，如果已有已核销排餐记录，接口返回错误，不更新日历状态。
- `additions` 必须绑定当前客户的进行中订单，且订单餐次类型需包含目标餐次。
- `excludedDates` 优先级高于 `additions`。
