# 客户用餐统计页面接口文档

## 接口概述

用于展示“客户用餐统计”页面数据，页面结构参考 `2025年客户用餐统计(2)_副本.xlsx` 第一个 sheet 的 A-K 列。

接口按“客户 + 餐次明细行”返回数据：

- 客户只有早餐：返回 1 行
- 客户只有午晚餐：返回 1 行
- 客户同时有早餐和午晚餐：返回 2 行

其中早餐行优先展示在前，午晚餐行展示在后。

---

## 请求地址

```http
GET /api/customerProfile/mealStats
```

---

## 权限

沿用客户档案列表权限：

```text
customerProfile:list
```

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `customerCode` | string | 否 | 客户编号，模糊匹配 |
| `customerName` | string | 否 | 客户姓名，模糊匹配 |
| `phone` | string | 否 | 手机号，模糊匹配 |
| `statsMonth` | string | 否 | 统计月份，格式 `yyyy-MM`。例如 `2026-04` 表示查询开始送餐时间早于 `2026-05-01` 且仍有剩余餐数的进行中订单客户 |
| `page` | int | 否 | 页码，从 1 开始，默认 1 |
| `size` | int | 否 | 每页条数，默认 10 |

## 统计口径

### 1. 客户筛选范围

- 先按客户编号、姓名、手机号筛选客户档案
- 再筛出存在进行中订单（`status = 1`）且订单剩余餐数 `remaining_count > 0` 的客户
- 传入 `statsMonth` 时，订单开始送餐时间 `start_date` 必须早于所选月份下一月月初
  - 例如 `statsMonth=2026-04` 时，筛选条件为 `start_date < 2026-05-01`
  - 其他月份同理，例如 `statsMonth=2026-05` 时，筛选条件为 `start_date < 2026-06-01`
  - 不按订单结束时间过滤

### 2. 行拆分规则

- `breakfast_count > 0`：生成早餐行
- `lunch_dinner_count > 0`：生成午晚餐行
- 同一客户两种餐池都存在时，返回两行

### 3. 餐数与剩余餐数

- 早餐行：
  - `mealCount = Σ breakfast_count`
  - `remainingMealCount = Σ breakfast_count - Σ BREAKFAST 已核销数`
- 午晚餐行：
  - `mealCount = Σ lunch_dinner_count`
  - `remainingMealCount = Σ lunch_dinner_count - Σ LUNCH 已核销数 - Σ DINNER 已核销数`

### 4. 地址展示

优先级：

```text
DEFAULT > WORKDAY > WEEKEND
```

展示格式：

```text
联系人：xxx
电话：xxx
地址：xxx
```

### 5. 送餐情况

- 早餐行固定显示 `早餐`
- 午晚餐行按 `schedule_mode + meal_type` 组合展示
  - 例：`DAILY + ALL` => `每日/午餐/晚餐`
  - 例：`DAILY + LUNCH` => `每日/午餐`
  - 例：`DAILY + DINNER` => `每日/晚餐`
