# 智能排查助手接口文档

## 1. 后台对前端接口

### 1.1 排餐未生成诊断

- Method: `POST`
- Path: `/api/agent/meal-plan/diagnose`
- 调用方: `eladmin-web`
- 提供方: `eladmin-system`
- 权限: `agentDiagnosis:list`

请求体:

```json
{
  "customerId": 1001,
  "customerCode": "C20260518001",
  "recordDate": "2026-05-17",
  "mealType": "LUNCH"
}
```

字段说明:

- `customerId`: 客户 ID，和 `customerCode` 至少传一个
- `customerCode`: 客户编号，和 `customerId` 至少传一个
- `recordDate`: 诊断日期，格式 `yyyy-MM-dd`
- `mealType`: 餐次，可选 `BREAKFAST`、`LUNCH`、`DINNER`

响应体:

```json
{
  "requestId": "diag-20260518-001",
  "customerId": 1001,
  "customerName": "张三",
  "recordDate": "2026-05-17",
  "mealType": "LUNCH",
  "summary": "客户 1001 在 2026-05-17 午餐未生成排餐，最可能原因是命中客户排除日期。",
  "ruleVersionDigest": "a4f8c1e9d320",
  "modelName": "spring-ai-chat-client",
  "fallback": false,
  "reasons": [
    {
      "code": "CUSTOMER_EXCLUDE_DATE_HIT",
      "title": "命中客户排除日期",
      "level": "HIGH",
      "description": "客户档案配置了目标日期和餐次不配送。",
      "suggestion": "先核对客户是否已登记停送。",
      "evidence": [
        {
          "label": "ruleId",
          "value": "CUSTOMER_EXCLUDE_DATE_HIT"
        }
      ]
    }
  ]
}
```

响应字段说明:

- `summary`: AI 诊断摘要，供客服快速阅读
- `ruleVersionDigest`: 本次使用的规则版本摘要，用于排查和审计
- `modelName`: 模型调用方标识；兜底结果可能为空
- `fallback`: 是否为兜底结果
- `reasons`: 可能原因列表
- `reasons[].code`: 稳定原因码
- `reasons[].level`: `HIGH`、`MEDIUM`、`LOW`
- `reasons[].evidence`: 字段级证据，必须能追溯到规则或上下文数据

## 2. 主系统到 agent-service 接口

### 2.1 agent-service 诊断接口

- Method: `POST`
- Path: `/api/agent/meal-plan/diagnose`
- 调用方: `eladmin-system`
- 提供方: `agent-service`

请求和响应结构与前端接口保持一致。

`eladmin-system` 不拼接 AI prompt，只做权限、审计、代理调用和兜底展示。

## 3. agent-service 到主系统内部接口

### 3.1 诊断上下文接口

- Method: `POST`
- Path: `/api/internal/agent/meal-plan/context`
- 调用方: `agent-service`
- 提供方: `eladmin-system`

请求体:

```json
{
  "customerId": 1001,
  "customerCode": "C20260518001",
  "recordDate": "2026-05-17",
  "mealType": "LUNCH"
}
```

响应包含:

- 客户档案
- 目标日期相关订单
- 排餐主记录
- 候选菜统计
- 客户排除日期、过敏、忌口等诊断上下文

## 4. 兜底策略

以下情况返回 `fallback=true`:

- `agent-service` 不可用
- AI 模型超时
- AI 输出不是合法结构
- AI 输出包含“已修复”“已修改数据库”等不允许的写操作描述
- rule registry 加载失败

兜底结果只提示人工核对方向，不给出确定业务结论。
